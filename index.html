API Reference

Core endpoints required for C4 Protocol compliance:

Meson (Compute) Endpoints

Endpoints for virtual machine management on the cloud provider. The matcher server listens on port 7777.

Status & Information

GET /api/v1.0/status

Returns details in plain text format about the Meson provider

Response includes:
  • Provider information
  • Supported services
  • Supported OS versions
  • VM configuration ranges
GET /api/v1.0/statusjson

Returns details in JSON format about the Meson provider

Response includes:
  • Provider information
  • Supported services
  • OS configurations with ISO templates
  • CPU and memory specifications

VM Management

GET /api/v1.0/running

List running instances with detailed status information

Response includes:
  • Instance metadata
  • Resource allocation
  • Network configuration
  • Billing information
GET /api/v1.0/canallocate

Check if a configuration can be allocated on the cloud provider

Response includes:
  • Allocation possibility
  • Pricing information (hourly/monthly)
POST /api/v1.0/register

Create new instances with specified configurations

Supports:
  • Custom resource allocation
  • Network configuration
  • Storage attachment
  • Authentication settings
  • Synchronous and asynchronous operations
DELETE /api/v1.0/unregister

Remove instances and cleanup resources

Features:
  • Graceful shutdown
  • Resource cleanup
  • Billing termination

Storage Endpoints

Endpoints for disk/volume management on the cloud provider. The storage server listens on port 7772.

Volume Management

POST /api/v1.0/create

Create a disk on the cloud provider

Configurable options:
  • Volume size
  • Bootable flag
  • Access permissions (private/public)
  • Read-only settings
  • Shareability between machines
POST /api/v1.0/info

Returns information about a disk by volume ID

POST /api/v1.0/accessible

Returns all visible disks for a client

Returns:
  • Private volumes created by the client
  • All public volumes
POST /api/v1.0/destroy

Delete a disk on the cloud provider

GET /api/v1.0/cancreate

Check if a volume configuration can be created

Response includes:
  • Creation possibility
  • Pricing information

Common API Features

Asynchronous Operations

Many endpoints support both synchronous and asynchronous operations via the async header:

  • When set to true, the response includes a poolingURL to check operation status
  • When set to false, the operation completes before responding

Consistent Response Format

All API responses follow a consistent JSON structure with:

  • Resource identifiers (UUIDs)
  • Resource metadata
  • Operation status
  • Error information when applicable

Integration Guide

Quick Start

Begin integration with our Docker-based development environment:

git clone --recurse-submodules http://github.com/elemento-modular-cloud/c4-protocol
docker compose up --build -d

Configuration Template

Essential provider configuration parameters:

auth:
    provider_id: 
    api_key: 
    endpoint: 
    region:

Additional configuration options available in the full documentation

Implementation Requirements

To implement the C4 Protocol, your service must provide:

  • REST API endpoints following the documented specifications
  • Support for both synchronous and asynchronous operations
  • Consistent error handling and status reporting
  • Resource lifecycle management (create, read, delete)

Testing Your Implementation

Verify your implementation with our compliance testing suite:

cd c4-protocol/tests
python -m pytest test_compliance.py --provider=
